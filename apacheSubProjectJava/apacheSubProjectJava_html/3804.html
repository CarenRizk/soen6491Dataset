149      * <h3>Reading from Cloud Spanner</h3>
150      *
151      * <h4>Bulk reading of a single query or table</h4>
152      *
153      * <p>To perform a single read from Cloud Spanner, construct a {@link Read} transform using {@link
154      * SpannerIO#read() SpannerIO.read()}. It will return a {@link PCollection} of {@link Struct
155      * Structs}, where each element represents an individual row returned from the read operation. Both
156      * Query and Read APIs are supported. See more information about <a
157      * href="https://cloud.google.com/spanner/docs/reads">reading from Cloud Spanner</a>
158      *
159      * <p>To execute a <strong>Query</strong>, specify a {@link Read#withQuery(Statement)} or {@link
160      * Read#withQuery(String)} during the construction of the transform.
161      *
162      * <pre>{@code
163      * PCollection<Struct> rows = p.apply(
164      *     SpannerIO.read()
165      *         .withInstanceId(instanceId)
166      *         .withDatabaseId(dbId)
167      *         .withQuery("SELECT id, name, email FROM users"));
168      * }</pre>
169      *
170      * <p>Reads by default use the <a
171      * href="https://cloud.google.com/spanner/docs/reads#read_data_in_parallel">PartitionQuery API</a>
172      * which enforces some limitations on the type of queries that can be used so that the data can be
173      * read in parallel. If the query is not supported by the PartitionQuery API, then you can specify a
174      * non-partitioned read by setting {@link Read#withBatching(boolean) withBatching(false)}. If the
175      * amount of data being read by a non-partitioned read is very large, it may be useful to add a
176      * {@link Reshuffle#viaRandomKey()} transform on the output so that the downstream transforms can
177      * run in parallel.
178      *
179      * <p>To read an entire <strong>Table</strong>, use {@link Read#withTable(String)} and optionally
180      * specify a {@link Read#withColumns(List) list of columns}.
181      *
182      * <pre>{@code
183      * PCollection<Struct> rows = p.apply(
184      *    SpannerIO.read()
185      *        .withInstanceId(instanceId)
186      *        .withDatabaseId(dbId)
187      *        .withTable("users")
188      *        .withColumns("id", "name", "email"));
189      * }</pre>
190      *
191      * <p>To read using an <strong>Index</strong>, specify the index name using {@link
192      * Read#withIndex(String)}.
193      *
194      * <pre>{@code
195      * PCollection<Struct> rows = p.apply(
196      *    SpannerIO.read()
197      *        .withInstanceId(instanceId)
198      *        .withDatabaseId(dbId)
199      *        .withTable("users")
200      *        .withIndex("users_by_name")
201      *        .withColumns("id", "name", "email"));
202      * }</pre>
203      *
204      * <h4>Read consistency</h4>
205      *
206      * <p>The transform is guaranteed to be executed on a consistent snapshot of data, utilizing the
207      * power of read only transactions. Staleness of data can be controlled using {@link
208      * Read#withTimestampBound} or {@link Read#withTimestamp(Timestamp)} methods. <a
209      * href="https://cloud.google.com/spanner/docs/transactions#read-only_transactions">Read more</a>
210      * about transactions in Cloud Spanner.
211      *
212      * <p>It is possible to read several {@link PCollection PCollections} within a single transaction.
213      * Apply {@link SpannerIO#createTransaction()} transform, that lazily creates a transaction. The
214      * result of this transformation can be passed to read operation using {@link
215      * Read#withTransaction(PCollectionView)}.
216      *
217      * <pre>{@code
218      * SpannerConfig spannerConfig = ...
219      *
220      * PCollectionView<Transaction> tx = p.apply(
221      *    SpannerIO.createTransaction()
222      *        .withSpannerConfig(spannerConfig)
223      *        .withTimestampBound(TimestampBound.strong()));
224      *
225      * PCollection<Struct> users = p.apply(
226      *    SpannerIO.read()
227      *        .withSpannerConfig(spannerConfig)
228      *        .withQuery("SELECT name, email FROM users")
229      *        .withTransaction(tx));
230      *
231      * PCollection<Struct> tweets = p.apply(
232      *    SpannerIO.read()
233      *        .withSpannerConfig(spannerConfig)
234      *        .withQuery("SELECT user, tweet, date FROM tweets")
235      *        .withTransaction(tx));
236      * }</pre>
237      *
238      * <h4>Bulk reading of multiple queries or tables</h4>
239      *
240      * You can perform multiple consistent reads on a set of tables or using a set of queries by
241      * constructing a {@link ReadAll} transform using {@link SpannerIO#readAll() SpannerIO.readAll()}.
242      * This transform takes a {@link PCollection} of {@link ReadOperation} elements, and performs the
243      * partitioned read on each of them using the same Read Only Transaction for consistent results.
244      *
245      * <p>Note that this transform should <strong>not</strong> be used in Streaming pipelines. This is
246      * because the same Read Only Transaction, which is created once when the pipeline is first
247      * executed, will be used for all reads. The data being read will therefore become stale, and if no
248      * reads are made for more than 1 hour, the transaction will automatically timeout and be closed by
249      * the Spanner server, meaning that any subsequent reads will fail.
250      *
251      * <pre>{@code
252      * // Build a collection of ReadOperations.
253      * PCollection<ReadOperation> reads = ...
254      *
255      * PCollection<Struct> rows = reads.apply(
256      *     SpannerIO.readAll()
257      *         .withInstanceId(instanceId)
258      *         .withDatabaseId(dbId)
259      * }</pre>
260      *
261      * <h3>Writing to Cloud Spanner</h3>
262      *
263      * <p>The Cloud Spanner {@link Write} transform writes to Cloud Spanner by executing a collection of
264      * input row {@link Mutation Mutations}. The mutations are grouped into batches for efficiency.
265      *
266      * <p>To configure the write transform, create an instance using {@link #write()} and then specify
267      * the destination Cloud Spanner instance ({@link Write#withInstanceId(String)} and destination
268      * database ({@link Write#withDatabaseId(String)}). For example:
269      *
270      * <pre>{@code
271      * // Earlier in the pipeline, create a PCollection of Mutations to be written to Cloud Spanner.
272      * PCollection<Mutation> mutations = ...;
273      * // Write mutations.
274      * SpannerWriteResult result = mutations.apply(
275      *     "Write", SpannerIO.write().withInstanceId("instance").withDatabaseId("database"));
276      * }</pre>
277      *
278      * <h3>SpannerWriteResult</h3>
279      *
280      * <p>The {@link SpannerWriteResult SpannerWriteResult} object contains the results of the
281      * transform, including a {@link PCollection} of MutationGroups that failed to write, and a {@link
282      * PCollection} that can be used in batch pipelines as a completion signal to {@link Wait
283      * Wait.OnSignal} to indicate when all input has been written. Note that in streaming pipelines,
284      * this signal will never be triggered as the input is unbounded and this {@link PCollection} is
285      * using the {@link GlobalWindow}.
286      *
287      * <h3>Batching and Grouping</h3>
288      *
289      * <p>To reduce the number of transactions sent to Spanner, the {@link Mutation Mutations} are
290      * grouped into batches. The default maximum size of the batch is set to 1MB or 5000 mutated cells,
291      * or 500 rows (whichever is reached first). To override this use {@link
292      * Write#withBatchSizeBytes(long) withBatchSizeBytes()}, {@link Write#withMaxNumMutations(long)
293      * withMaxNumMutations()} or {@link Write#withMaxNumMutations(long) withMaxNumRows()}. Setting
294      * either to a small value or zero disables batching.
295      *
296      * <p>Note that the <a
297      * href="https://cloud.google.com/spanner/quotas#limits_for_creating_reading_updating_and_deleting_data">maximum
298      * size of a single transaction</a> is 20,000 mutated cells - including cells in indexes. If you
299      * have a large number of indexes and are getting exceptions with message: <tt>INVALID_ARGUMENT: The
300      * transaction contains too many mutations</tt> you will need to specify a smaller number of {@code
301      * MaxNumMutations}.
302      *
303      * <p>The batches written are obtained from by grouping enough {@link Mutation Mutations} from the
304      * Bundle provided by Beam to form several batches. This group of {@link Mutation Mutations} is then
305      * sorted by table and primary key, and the batches are created from the sorted group. Each batch
306      * will then have rows for the same table, with keys that are 'close' to each other, thus optimising
307      * write efficiency by each batch affecting as few table splits as possible performance.
308      *
309      * <p>This grouping factor (number of batches) is controlled by the parameter {@link
310      * Write#withGroupingFactor(int) withGroupingFactor()}.
311      *
312      * <p>Note that each worker will need enough memory to hold {@code GroupingFactor x
313      * MaxBatchSizeBytes} Mutations, so if you have a large {@code MaxBatchSize} you may need to reduce
314      * {@code GroupingFactor}
315      *
316      * <p>While Grouping and Batching increases write efficiency, it dramatically increases the latency
317      * between when a Mutation is received by the transform, and when it is actually written to the
318      * database. This is because enough Mutations need to be received to fill the grouped batches. In
319      * Batch pipelines (bounded sources), this is not normally an issue, but in Streaming (unbounded)
320      * pipelines, this latency is often seen as unacceptable.
321      *
322      * <p>There are therefore 3 different ways that this transform can be configured:
323      *
324      * <ul>
325      *   <li>With Grouping and Batching. <br>
326      *       This is the default for Batch pipelines, where sorted batches of Mutations are created and
327      *       written. This is the most efficient way to ingest large amounts of data, but the highest
328      *       latency before writing
329      *   <li>With Batching but no Grouping <br>
330      *       If {@link Write#withGroupingFactor(int) .withGroupingFactor(1)}, is set, grouping is
331      *       disabled. This is the default for Streaming pipelines. Unsorted batches are created and
332      *       written as soon as enough mutations to fill a batch are received. This reflects a
333      *       compromise where a small amount of additional latency enables more efficient writes
334      *   <li>Without any Batching <br>
335      *       If {@link Write#withBatchSizeBytes(long) .withBatchSizeBytes(0)} is set, no batching is
336      *       performed and the Mutations are written to the database as soon as they are received.
337      *       ensuring the lowest latency before Mutations are written.
338      * </ul>
339      *
340      * <h3>Monitoring</h3>
341      *
342      * <p>Several counters are provided for monitoring purpooses:
343      *
344      * <ul>
345      *   <li><tt>batchable_mutation_groups</tt><br>
346      *       Counts the mutations that are batched for writing to Spanner.
347      *   <li><tt>unbatchable_mutation_groups</tt><br>
348      *       Counts the mutations that can not be batched and are applied individually - either because
349      *       they are too large to fit into a batch, or they are ranged deletes.
350      *   <li><tt>mutation_group_batches_received, mutation_group_batches_write_success,
351      *       mutation_group_batches_write_failed</tt><br>
352      *       Count the number of batches that are processed. If Failure Mode is set to {@link
353      *       FailureMode#REPORT_FAILURES REPORT_FAILURES}, then failed batches will be split up and the
354      *       individual mutation groups retried separately.
355      *   <li><tt>mutation_groups_received, mutation_groups_write_success,
356      *       mutation_groups_write_fail</tt><br>
357      *       Count the number of individual MutationGroups that are processed.
358      *   <li><tt>spanner_write_success, spanner_write_fail</tt><br>
359      *       The number of writes to Spanner that have occurred.
360      *   <li><tt>spanner_write_retries</tt><br>
361      *       The number of times a write is retried after a failure - either due to a timeout, or when
362      *       batches fail and {@link FailureMode#REPORT_FAILURES REPORT_FAILURES} is set so that
363      *       individual Mutation Groups are retried.
364      *   <li><tt>spanner_write_timeouts</tt><br>
365      *       The number of timeouts that occur when writing to Spanner. Writes that timed out are
366      *       retried after a backoff. Large numbers of timeouts suggest an overloaded Spanner instance.
367      *   <li><tt>spanner_write_total_latency_ms</tt><br>
368      *       The total amount of time spent writing to Spanner, in milliseconds.
369      * </ul>
370      *
371      * <h3>Database Schema Preparation</h3>
372      *
373      * <p>The Write transform reads the database schema on pipeline start to know which columns are used
374      * as primary keys of the tables and indexes. This is so that the transform knows how to sort the
375      * grouped Mutations by table name and primary key as described above.
376      *
377      * <p>If the database schema, any additional tables or indexes are created in the same pipeline then
378      * there will be a race condition, leading to a situation where the schema is read before the table
379      * is created its primary key will not be known. This will mean that the sorting/batching will not
380      * be optimal and performance will be reduced (warnings will be logged for rows using unknown
381      * tables)
382      *
383      * <p>To prevent this race condition, use {@link Write#withSchemaReadySignal(PCollection)} to pass a
384      * signal {@link PCollection} (for example the output of the transform that creates the table(s))
385      * which will be used with {@link Wait.OnSignal} to prevent the schema from being read until it is
386      * ready. The Write transform will be paused until this signal {@link PCollection} is closed.
387      *
388      * <h3>Transactions</h3>
389      *
390      * <p>The transform does not provide same transactional guarantees as Cloud Spanner. In particular,
391      *
392      * <ul>
393      *   <li>Individual Mutations are submitted atomically, but all Mutations are not submitted in the
394      *       same transaction.
395      *   <li>A Mutation is applied at least once;
396      *   <li>If the pipeline was unexpectedly stopped, mutations that were already applied will not get
397      *       rolled back.
398      * </ul>
399      *
400      * <p>Use {@link MutationGroup MutationGroups} with the {@link WriteGrouped} transform to ensure
401      * that a small set mutations is bundled together. It is guaranteed that mutations in a {@link
402      * MutationGroup} are submitted in the same transaction. Note that a MutationGroup must not exceed
403      * the Spanner transaction limits.
404      *
405      * <pre>{@code
406      * // Earlier in the pipeline, create a PCollection of MutationGroups to be written to Cloud Spanner.
407      * PCollection<MutationGroup> mutationGroups = ...;
408      * // Write mutation groups.
409      * SpannerWriteResult result = mutationGroups.apply(
410      *     "Write",
411      *     SpannerIO.write().withInstanceId("instance").withDatabaseId("database").grouped());
412      * }</pre>
413      *
414      * <h3>Streaming Support</h3>
415      *
416      * <p>{@link Write} can be used as a streaming sink, however as with batch mode note that the write
417      * order of individual {@link Mutation}/{@link MutationGroup} objects is not guaranteed.
418      *
419      * <p>{@link Read} and {@link ReadAll} can be used in Streaming pipelines to read a set of Facts on
420      * pipeline startup.
421      *
422      * <p>{@link ReadAll} should not be used on an unbounded {@code PCollection<ReadOperation>}, for the
423      * reasons stated above.

1401       /** Same as {@link Write} but supports grouped mutations. */
1402       public static class WriteGrouped
1403           extends PTransform<PCollection<MutationGroup>, SpannerWriteResult> {
1404     
1405         private final Write spec;
1406         private static final TupleTag<MutationGroup> BATCHABLE_MUTATIONS_TAG =
1407             new TupleTag<MutationGroup>("batchableMutations") {};
1408         private static final TupleTag<Iterable<MutationGroup>> UNBATCHABLE_MUTATIONS_TAG =
1409             new TupleTag<Iterable<MutationGroup>>("unbatchableMutations") {};
1410     
1411         private static final TupleTag<Void> MAIN_OUT_TAG = new TupleTag<Void>("mainOut") {};
1412         private static final TupleTag<MutationGroup> FAILED_MUTATIONS_TAG =
1413             new TupleTag<MutationGroup>("failedMutations") {};
1414         private static final SerializableCoder<MutationGroup> CODER =
1415             SerializableCoder.of(MutationGroup.class);
1416     
1417         public WriteGrouped(Write spec) {
1418           this.spec = spec;
1419         }
1420     
1421         @Override
1422         public void populateDisplayData(DisplayData.Builder builder) {
1423           super.populateDisplayData(builder);
1424           spec.populateDisplayDataWithParamaters(builder);
1425         }
1426     
1427         @Override
1428         public SpannerWriteResult expand(PCollection<MutationGroup> input) {
1429           PCollection<Iterable<MutationGroup>> batches;
1430           PCollectionView<Dialect> dialectView = spec.getDialectView();
1431     
1432           if (dialectView == null) {
1433             dialectView =
1434                 input
1435                     .getPipeline()
1436                     .apply("CreateSingleton", Create.of(Dialect.GOOGLE_STANDARD_SQL))
1437                     .apply("As PCollectionView", View.asSingleton());
1438           }
1439     
1440           if (spec.getBatchSizeBytes() <= 1
1441               || spec.getMaxNumMutations() <= 1
1442               || spec.getMaxNumRows() <= 1) {
1443             LOG.info("Batching of mutationGroups is disabled");
1444             TypeDescriptor<Iterable<MutationGroup>> descriptor =
1445                 new TypeDescriptor<Iterable<MutationGroup>>() {};
1446             batches = input.apply(MapElements.into(descriptor).via(ImmutableList::of));
1447           } else {
1448     
1449             // First, read the Cloud Spanner schema.
1450             PCollection<Void> schemaSeed =
1451                 input.getPipeline().apply("Create Seed", Create.of((Void) null));
1452             if (spec.getSchemaReadySignal() != null) {
1453               // Wait for external signal before reading schema.
1454               schemaSeed = schemaSeed.apply("Wait for schema", Wait.on(spec.getSchemaReadySignal()));
1455             }
1456             final PCollectionView<SpannerSchema> schemaView =
1457                 schemaSeed
1458                     .apply(
1459                         "Read information schema",
1460                         ParDo.of(new ReadSpannerSchema(spec.getSpannerConfig(), dialectView))
1461                             .withSideInputs(dialectView))
1462                     .apply("Schema View", View.asSingleton());
1463     
1464             // Split the mutations into batchable and unbatchable mutations.
1465             // Filter out mutation groups too big to be batched.
1466             PCollectionTuple filteredMutations =
1467                 input
1468                     .apply(
1469                         "RewindowIntoGlobal",
1470                         Window.<MutationGroup>into(new GlobalWindows())
1471                             .triggering(DefaultTrigger.of())
1472                             .discardingFiredPanes())
1473                     .apply(
1474                         "Filter Unbatchable Mutations",
1475                         ParDo.of(
1476                                 new BatchableMutationFilterFn(
1477                                     schemaView,
1478                                     UNBATCHABLE_MUTATIONS_TAG,
1479                                     spec.getBatchSizeBytes(),
1480                                     spec.getMaxNumMutations(),
1481                                     spec.getMaxNumRows()))
1482                             .withSideInputs(schemaView)
1483                             .withOutputTags(
1484                                 BATCHABLE_MUTATIONS_TAG, TupleTagList.of(UNBATCHABLE_MUTATIONS_TAG)));
1485     
1486             // Build a set of Mutation groups from the current bundle,
1487             // sort them by table/key then split into batches.
1488             PCollection<Iterable<MutationGroup>> batchedMutations =
1489                 filteredMutations
1490                     .get(BATCHABLE_MUTATIONS_TAG)
1491                     .apply(
1492                         "Gather Sort And Create Batches",
1493                         ParDo.of(
1494                                 new GatherSortCreateBatchesFn(
1495                                     spec.getBatchSizeBytes(),
1496                                     spec.getMaxNumMutations(),
1497                                     spec.getMaxNumRows(),
1498                                     // Do not group on streaming unless explicitly set.
1499                                     spec.getGroupingFactor()
1500                                         .orElse(
1501                                             input.isBounded() == IsBounded.BOUNDED
1502                                                 ? DEFAULT_GROUPING_FACTOR
1503                                                 : 1),
1504                                     schemaView))
1505                             .withSideInputs(schemaView));
1506     
1507             // Merge the batched and unbatchable mutation PCollections and write to Spanner.
1508             batches =
1509                 PCollectionList.of(filteredMutations.get(UNBATCHABLE_MUTATIONS_TAG))
1510                     .and(batchedMutations)
1511                     .apply("Merge", Flatten.pCollections());
1512           }
1513     
1514           PCollectionTuple result =
1515               batches.apply(
1516                   "Write batches to Spanner",
1517                   ParDo.of(
1518                           new WriteToSpannerFn(
1519                               spec.getSpannerConfig(), spec.getFailureMode(), FAILED_MUTATIONS_TAG))
1520                       .withOutputTags(MAIN_OUT_TAG, TupleTagList.of(FAILED_MUTATIONS_TAG)));
1521     
1522           return new SpannerWriteResult(
1523               input.getPipeline(),
1524               result.get(MAIN_OUT_TAG),
1525               result.get(FAILED_MUTATIONS_TAG),
1526               FAILED_MUTATIONS_TAG);
1527         }

1893        * outputs the encoded sorted list.
1894        *
1895        * <p>Testing notes: With very small amounts of data, each mutation group is in a separate bundle,
1896        * and as batching and sorting is over the bundle, this effectively means that no batching will
1897        * occur, Therefore this DoFn has to be tested in isolation.
1898        */
1899       @VisibleForTesting
1900       static class GatherSortCreateBatchesFn extends DoFn<MutationGroup, Iterable<MutationGroup>> {
1901     
1902         private final long maxBatchSizeBytes;
1903         private final long maxBatchNumMutations;
1904         private final long maxBatchNumRows;
1905         private final long maxSortableSizeBytes;
1906         private final long maxSortableNumMutations;
1907         private final long maxSortableNumRows;
1908         private final PCollectionView<SpannerSchema> schemaView;
1909         private final ArrayList<MutationGroupContainer> mutationsToSort = new ArrayList<>();
1910     
1911         // total size of MutationGroups in mutationsToSort.
1912         private long sortableSizeBytes = 0;
1913         // total number of mutated cells in mutationsToSort
1914         private long sortableNumCells = 0;
1915         // total number of rows mutated in mutationsToSort
1916         private long sortableNumRows = 0;
1917     
1918         GatherSortCreateBatchesFn(
1919             long maxBatchSizeBytes,
1920             long maxNumMutations,
1921             long maxNumRows,
1922             long groupingFactor,
1923             PCollectionView<SpannerSchema> schemaView) {
1924           this.maxBatchSizeBytes = maxBatchSizeBytes;
1925           this.maxBatchNumMutations = maxNumMutations;
1926           this.maxBatchNumRows = maxNumRows;
1927     
1928           if (groupingFactor <= 0) {
1929             groupingFactor = 1;
1930           }
1931     
1932           this.maxSortableSizeBytes = maxBatchSizeBytes * groupingFactor;
1933           this.maxSortableNumMutations = maxNumMutations * groupingFactor;
1934           this.maxSortableNumRows = maxNumRows * groupingFactor;
1935           this.schemaView = schemaView;
1936     
1937           initSorter();
1938         }

1940         private synchronized void initSorter() {
1941           mutationsToSort.clear();
1942           sortableSizeBytes = 0;
1943           sortableNumCells = 0;
1944           sortableNumRows = 0;
1945         }

1947         @FinishBundle
1948         public synchronized void finishBundle(FinishBundleContext c) throws Exception {
1949           sortAndOutputBatches(new OutputReceiverForFinishBundle(c));

1952         private synchronized void sortAndOutputBatches(OutputReceiver<Iterable<MutationGroup>> out)
1953             throws IOException {
1954           try {
1955             if (mutationsToSort.isEmpty()) {
1956               // nothing to output.
1957               return;
1958             }
1959     
1960             if (maxSortableNumMutations == maxBatchNumMutations) {
1961               // no grouping is occurring, no need to sort and make batches, just output what we have.
1962               outputBatch(out, 0, mutationsToSort.size());
1963               return;
1964             }
1965     
1966             // Sort then split the sorted mutations into batches.
1967             mutationsToSort.sort(Comparator.naturalOrder());
1968             int batchStart = 0;
1969             int batchEnd = 0;
1970     
1971             // total size of the current batch.
1972             long batchSizeBytes = 0;
1973             // total number of mutated cells.
1974             long batchCells = 0;
1975             // total number of rows mutated.
1976             long batchRows = 0;
1977     
1978             // collect and output batches.
1979             while (batchEnd < mutationsToSort.size()) {
1980               MutationGroupContainer mg = mutationsToSort.get(batchEnd);
1981     
1982               if (((batchCells + mg.numCells) > maxBatchNumMutations)
1983                   || ((batchSizeBytes + mg.sizeBytes) > maxBatchSizeBytes
1984                       || (batchRows + mg.numRows > maxBatchNumRows))) {
1985                 // Cannot add new element, current batch is full; output.
1986                 outputBatch(out, batchStart, batchEnd);
1987                 batchStart = batchEnd;
1988                 batchSizeBytes = 0;
1989                 batchCells = 0;
1990                 batchRows = 0;
1991               }
1992     
1993               batchEnd++;
1994               batchSizeBytes += mg.sizeBytes;
1995               batchCells += mg.numCells;
1996               batchRows += mg.numRows;
1997             }
1998     
1999             if (batchStart < batchEnd) {
2000               // output remaining elements
2001               outputBatch(out, batchStart, mutationsToSort.size());
2002             }
2003           } finally {
2004             initSorter();
2005           }
2006         }
2007     
2008         private void outputBatch(
2009             OutputReceiver<Iterable<MutationGroup>> out, int batchStart, int batchEnd) {
2010           out.output(
2011               mutationsToSort.subList(batchStart, batchEnd).stream()
2012                   .map(o -> o.mutationGroup)
2013                   .collect(toList()));
2014         }

2016         @ProcessElement
2017         public synchronized void processElement(
2018             ProcessContext c, OutputReceiver<Iterable<MutationGroup>> out) throws Exception {
2019           SpannerSchema spannerSchema = c.sideInput(schemaView);
2020           MutationKeyEncoder encoder = new MutationKeyEncoder(spannerSchema);
2021           MutationGroup mg = c.element();
2022           long groupSize = MutationSizeEstimator.sizeOf(mg);
2023           long groupCells = MutationCellCounter.countOf(spannerSchema, mg);
2024           long groupRows = mg.size();
2025     
2026           synchronized (this) {
2027             if (((sortableNumCells + groupCells) > maxSortableNumMutations)
2028                 || (sortableSizeBytes + groupSize) > maxSortableSizeBytes
2029                 || (sortableNumRows + groupRows) > maxSortableNumRows) {
2030               sortAndOutputBatches(out);
2031             }
2032     
2033             mutationsToSort.add(
2034                 new MutationGroupContainer(
2035                     mg, groupSize, groupCells, groupRows, encoder.encodeTableNameAndKey(mg.primary())));
2036             sortableSizeBytes += groupSize;
2037             sortableNumCells += groupCells;
2038             sortableNumRows += groupRows;
2039           }
2040         }

2319         /** Write the Mutations to Spanner, handling DEADLINE_EXCEEDED with backoff/retries. */
2320         private void writeMutations(Iterable<Mutation> mutationIterable)
2321             throws SpannerException, IOException {
2322           BackOff backoff = bundleWriteBackoff.backoff();
2323           List<Mutation> mutations = ImmutableList.copyOf(mutationIterable);
2324     
2325           while (true) {
2326             Stopwatch timer = Stopwatch.createStarted();
2327             // loop is broken on success, timeout backoff/retry attempts exceeded, or other failure.
2328             try {
2329               spannerWriteSuccess.inc();
2330               return;
2331             } catch (SpannerException exception) {
2332               if (exception.getErrorCode() == ErrorCode.DEADLINE_EXCEEDED) {
2333                 spannerWriteTimeouts.inc();
2334     
2335                 // Potentially backoff/retry after DEADLINE_EXCEEDED.
2336                 long sleepTimeMsecs = backoff.nextBackOffMillis();
2337                 if (sleepTimeMsecs == BackOff.STOP) {
2338                   LOG.error(
2339                       "DEADLINE_EXCEEDED writing batch of {} mutations to Cloud Spanner. "
2340                           + "Aborting after too many retries.",
2341                       mutations.size());
2342                   spannerWriteFail.inc();
2343                   throw exception;
2344                 }
2345                 LOG.info(
2346                     "DEADLINE_EXCEEDED writing batch of {} mutations to Cloud Spanner, "
2347                         + "retrying after backoff of {}ms\n"
2348                         + "({})",
2349                     mutations.size(),
2350                     sleepTimeMsecs,
2351                     exception.getMessage());
2352                 spannerWriteRetries.inc();
2353                 try {
