37     /**
38      * {@link Window} logically divides up or groups the elements of a {@link PCollection} into finite
39      * windows according to a {@link WindowFn}. The output of {@code Window} contains the same elements
40      * as input, but they have been logically assigned to windows. The next {@link
41      * org.apache.beam.sdk.transforms.GroupByKey GroupByKeys}, including one within composite
42      * transforms, will group by the combination of keys and windows.
43      *
44      * <p>See {@link org.apache.beam.sdk.transforms.GroupByKey} for more information about how grouping
45      * with windows works.
46      *
47      * <h2>Windowing</h2>
48      *
49      * <p>Windowing a {@link PCollection} divides the elements into windows based on the associated
50      * event time for each element. This is especially useful for {@link PCollection PCollections} with
51      * unbounded size, since it allows operating on a sub-group of the elements placed into a related
52      * window. For {@link PCollection PCollections} with a bounded size (aka. conventional batch mode),
53      * by default, all data is implicitly in a single window, unless {@link Window} is applied.
54      *
55      * <p>For example, a simple form of windowing divides up the data into fixed-width time intervals,
56      * using {@link FixedWindows}. The following example demonstrates how to use {@link Window} in a
57      * pipeline that counts the number of occurrences of strings each minute:
58      *
59      * <pre>{@code
60      * PCollection<String> items = ...;
61      * PCollection<String> windowed_items = items.apply(
62      *   Window.<String>into(FixedWindows.of(Duration.standardMinutes(1))));
63      * PCollection<KV<String, Long>> windowed_counts = windowed_items.apply(
64      *   Count.<String>perElement());
65      * }</pre>
66      *
67      * <p>Let (data, timestamp) denote a data element along with its timestamp. Then, if the input to
68      * this pipeline consists of {("foo", 15s), ("bar", 30s), ("foo", 45s), ("foo", 1m30s)}, the output
69      * will be {(KV("foo", 2), 1m), (KV("bar", 1), 1m), (KV("foo", 1), 2m)}
70      *
71      * <p>Several predefined {@link WindowFn}s are provided:
72      *
73      * <ul>
74      *   <li>{@link FixedWindows} partitions the timestamps into fixed-width intervals.
75      *   <li>{@link SlidingWindows} places data into overlapping fixed-width intervals.
76      *   <li>{@link Sessions} groups data into sessions where each item in a window is separated from
77      *       the next by no more than a specified gap.
78      * </ul>
79      *
80      * <p>Additionally, custom {@link WindowFn}s can be created, by creating new subclasses of {@link
81      * WindowFn}.
82      *
83      * <h2>Triggers</h2>
84      *
85      * <p>{@link Window#triggering(Trigger)} allows specifying a trigger to control when (in processing
86      * time) results for the given window can be produced. If unspecified, the default behavior is to
87      * trigger first when the watermark passes the end of the window, and then trigger again every time
88      * there is late arriving data.
89      *
90      * <p>Elements are added to the current window pane as they arrive. When the root trigger fires,
91      * output is produced based on the elements in the current pane.
92      *
93      * <p>Depending on the trigger, this can be used both to output partial results early during the
94      * processing of the whole window, and to deal with late arriving in batches.
95      *
96      * <p>Continuing the earlier example, if we wanted to emit the values that were available when the
97      * watermark passed the end of the window, and then output any late arriving elements once-per
98      * (actual hour) hour until we have finished processing the next 24-hours of data. (The use of
99      * watermark time to stop processing tends to be more robust if the data source is slow for a few
100      * days, etc.)
101      *
102      * <pre>{@code
103      * PCollection<String> items = ...;
104      * PCollection<String> windowed_items = items.apply(
105      *   Window.<String>into(FixedWindows.of(Duration.standardMinutes(1)))
106      *      .triggering(
107      *          AfterWatermark.pastEndOfWindow()
108      *              .withLateFirings(AfterProcessingTime
109      *                  .pastFirstElementInPane().plusDelayOf(Duration.standardHours(1))))
110      *      .withAllowedLateness(Duration.standardDays(1)));
111      * PCollection<KV<String, Long>> windowed_counts = windowed_items.apply(
112      *   Count.<String>perElement());
113      * }</pre>
114      *
115      * <p>On the other hand, if we wanted to get early results every minute of processing time (for
116      * which there were new elements in the given window) we could do the following:
117      *
118      * <pre>{@code
119      * PCollection<String> windowed_items = items.apply(
120      *   Window.<String>into(FixedWindows.of(Duration.standardMinutes(1)))
121      *      .triggering(
122      *          AfterWatermark.pastEndOfWindow()
123      *              .withEarlyFirings(AfterProcessingTime
124      *                  .pastFirstElementInPane().plusDelayOf(Duration.standardMinutes(1))))
125      *      .withAllowedLateness(Duration.ZERO));
126      * }</pre>
127      *
128      * <p>After a {@link org.apache.beam.sdk.transforms.GroupByKey} the trigger is set to a trigger that
129      * will preserve the intent of the upstream trigger. See {@link Trigger#getContinuationTrigger} for
130      * more information.
131      *
132      * <p>See {@link Trigger} for details on the available triggers.
133      */

175        * Creates a {@code Window} {@code PTransform} that uses the given {@link WindowFn} to window the
176        * data.
177        *
178        * <p>The resulting {@code PTransform}'s types have been bound, with both the input and output
179        * being a {@code PCollection<T>}, inferred from the types of the argument {@code WindowFn}. It is

182       public static <T> Window<T> into(WindowFn<? super T, ?> fn) {
183         try {
184           fn.windowCoder().verifyDeterministic();
185         } catch (NonDeterministicException e) {
186           throw new IllegalArgumentException("Window coders must be deterministic.", e);
187         }
188         return Window.<T>configure().withWindowFn(fn);
189       }

322       /** Get the output strategy of this {@link Window Window PTransform}. For internal use only. */
323       public WindowingStrategy<?, ?> getOutputStrategyInternal(WindowingStrategy<?, ?> inputStrategy) {
324         WindowingStrategy<?, ?> result = inputStrategy;
325         if (getWindowFn() != null) {
326           result = result.withAlreadyMerged(false).withWindowFn(getWindowFn());
327         }
328         if (getTrigger() != null) {
329           result = result.withTrigger(getTrigger());
330         }
331         if (getAccumulationMode() != null) {
332           result = result.withMode(getAccumulationMode());
333         }
334         if (getAllowedLateness() != null) {
335           result =
336               result.withAllowedLateness(
337                   Ordering.natural().max(getAllowedLateness(), inputStrategy.getAllowedLateness()));
338         }
339         if (getClosingBehavior() != null) {
340           result = result.withClosingBehavior(getClosingBehavior());
341         }
342         if (getOnTimeBehavior() != null) {
343           result = result.withOnTimeBehavior(getOnTimeBehavior());
344         }
345         if (getTimestampCombiner() != null) {
346           result = result.withTimestampCombiner(getTimestampCombiner());
347         }
348         return result;
349       }

351       private void applicableTo(PCollection<?> input) {
352         WindowingStrategy<?, ?> outputStrategy =
353             getOutputStrategyInternal(input.getWindowingStrategy());
354     
355         // Make sure that the windowing strategy is complete & valid.
356         if (outputStrategy.isTriggerSpecified()
357             && !(outputStrategy.getTrigger() instanceof DefaultTrigger)
358             && !(outputStrategy.getWindowFn() instanceof GlobalWindows)
359             && !outputStrategy.isAllowedLatenessSpecified()) {
360           throw new IllegalArgumentException(
361               "Except when using GlobalWindows,"
362                   + " calling .triggering() to specify a trigger requires that the allowed lateness"
363                   + " be specified using .withAllowedLateness() to set the upper bound on how late"
364                   + " data can arrive before being dropped. See Javadoc for more details.");
365         }
366     
367         if (!outputStrategy.isModeSpecified() && canProduceMultiplePanes(outputStrategy)) {
368           throw new IllegalArgumentException(
369               "Calling .triggering() to specify a trigger or calling .withAllowedLateness() to"
370                   + " specify an allowed lateness greater than zero requires that the accumulation"
371                   + " mode be specified using .discardingFiredPanes() or .accumulatingFiredPanes()."
372                   + " See Javadoc for more details.");
373         }
374       }

376       private boolean canProduceMultiplePanes(WindowingStrategy<?, ?> strategy) {
377         // The default trigger is Repeatedly.forever(AfterWatermark.pastEndOfWindow()); This fires
378         // for every late-arriving element if allowed lateness is nonzero, and thus we must have
379         // an accumulating mode specified
380         boolean dataCanArriveLate =
381             !(strategy.getWindowFn() instanceof GlobalWindows)
382                 && strategy.getAllowedLateness().getMillis() > 0;
383         boolean hasCustomTrigger = !(strategy.getTrigger() instanceof DefaultTrigger);
384         return dataCanArriveLate || hasCustomTrigger;
385       }

387       @Override
388       public PCollection<T> expand(PCollection<T> input) {
389         applicableTo(input);
390     
391         WindowingStrategy<?, ?> outputStrategy =
392             getOutputStrategyInternal(input.getWindowingStrategy());
393     
394         if (getWindowFn() == null) {
395           // A new PCollection must be created in case input is reused in a different location as the
396           // two PCollections will, in general, have a different windowing strategy.
397           return PCollectionList.of(input)
398               .apply(Flatten.pCollections())
399               .setWindowingStrategyInternal(outputStrategy);
400         } else {
401           // This is the AssignWindows primitive
402           return input.apply(new Assign<>(this, outputStrategy));
403         }
404       }

453        * A Primitive {@link PTransform} that assigns windows to elements based on a {@link WindowFn}.
454        * Pipeline authors should use {@link Window} directly instead.
455        */
456       public static class Assign<T> extends PTransform<PCollection<T>, PCollection<T>> {
457         private final @Nullable Window<T> original;
458         private final WindowingStrategy<T, ?> updatedStrategy;
459     
460         /**
461          * Create a new {@link Assign} where the output is windowed with the updated {@link
462          * WindowingStrategy}. Windows should be assigned using the {@link WindowFn} returned by {@link
463          * #getWindowFn()}.
464          */
465         @VisibleForTesting
466         Assign(@Nullable Window<T> original, WindowingStrategy updatedStrategy) {
467           this.original = original;
468           this.updatedStrategy = updatedStrategy;
469         }
470     
471         @Override
472         public PCollection<T> expand(PCollection<T> input) {
473           return PCollection.createPrimitiveOutputInternal(
474               input.getPipeline(), updatedStrategy, input.isBounded(), input.getCoder());
475         }
476     
477         @Override
478         public void populateDisplayData(DisplayData.Builder builder) {
479           if (original != null) {
480             original.populateDisplayData(builder);
481           }
482         }
483     
484         public @Nullable WindowFn<T, ?> getWindowFn() {
485           return updatedStrategy.getWindowFn();
486         }
487     
488         public static <T> Assign<T> createInternal(WindowingStrategy finalStrategy) {
489           return new Assign<T>(null, finalStrategy);
490         }
491       }

502       /**
503        * {@code PTransform} that does not change assigned windows, but will cause windows to be merged
504        * again as part of the next {@link org.apache.beam.sdk.transforms.GroupByKey}.
505        */
506       private static class Remerge<T> extends PTransform<PCollection<T>, PCollection<T>> {
507         @Override
508         public PCollection<T> expand(PCollection<T> input) {
509           return input
510               // We first apply a (trivial) transform to the input PCollection to produce a new
511               // PCollection. This ensures that we don't modify the windowing strategy of the input
512               // which may be used elsewhere.
513               .apply(
514                   "Identity",
515                   MapElements.via(
516                       new SimpleFunction<T, T>() {
517                         @Override
518                         public T apply(T element) {
519                           return element;
520                         }
521                       }))
522               // Then we modify the windowing strategy.
523               .setWindowingStrategyInternal(input.getWindowingStrategy().withAlreadyMerged(false));
524         }
525       }
